Back in 90s JavaScript was a interpreted language. But in 2008 when google chrome launched their V8 engine which overcome the demerits of interpreted and compiled language. It actually combine the two majore facility of those language which are easy to debug and fast process which is known as just in time(JIT) compiler. JavaScript is a highlevel single threaded programming language. Whenever a javascript program runs inside a web browser, it transfered to the browser's engine. Then the engine compile the code and transform it into a machine-readable form which is known as machine language(binary language). Then the machine shows the output of the code in the interface. As it also gives us the facility of an interpreted language, that means the JIT compile the bytecode into the machine code of the running machine. Which gives us the facilities of platform independency as the interpreters execute the code itself and efficiency of running object code will overcome the inefficiency of recompiling the code every time it runs.

Event loop is the procidure which monitor the call stack and the callback queue. We all know that JavaScript is a single threaded language and it always operates it work in synchronous way. But it gives us the asynchronous behaviour by event loop. When any asynchronous function is called it transfered to the callback queue and wait till the synchronous function ends. When call stack is empty event loop gets executed and passes the first funcion in the call stack from the queue. Thus it manages the asynchronous behaviour of JavaScript.
